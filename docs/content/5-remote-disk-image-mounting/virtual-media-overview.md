# Virtual Media Overview

[cite_start]The **Virtual Media** subsystem enables the remote, out-of-band attachment of ISO images and raw disk file systems directly to the target server[cite: 25]. [cite_start]By emulating a physical USB mass storage device or CD/DVD-ROM drive at the hardware level [cite: 25, 26][cite_start], this architecture provides seamless pre-OS media access without requiring a complex, dependency-heavy PXE network boot infrastructure[cite: 24].

---

## 1. Core Capabilities & Workflows

The system supports highly flexible media workflows specifically engineered for bare-metal provisioning, automated staging, and remote disaster recovery scenarios:

* [cite_start]**Bare-Metal ISO Mounting:** Attach bootable disk images directly to execute zero-touch operating system installations or deploy low-level, isolated diagnostic utilities.
* [cite_start]**Flexible Source Selection:** Provision image files directly from the appliance's local high-endurance storage repository (MicroSD card / internal cache) or stream them dynamically from a remote administrative workstation via the native client application.
* [cite_start]**Universal Hardware-Level Emulation:** The target host's USB controller recognizes the attached virtual media as a standard, physically connected USB block device[cite: 25, 26]. [cite_start]This ensures universal plug-and-play compatibility across both legacy text-based BIOS and modern UEFI environments without requiring any custom host-level drivers[cite: 26].

---

## 2. Media Provisioning & Staging Workflow

[cite_start]From the perspective of the target server, the virtual mounting process is completely transparent and perfectly mimics the physical insertion of a physical USB flash drive or external optical drive[cite: 26]. 

To mount an image, execute the following three-step workflow:

1. **Image Staging:** Transfer the required `.iso` or raw disk image file onto the isolated USBridge local storage matrix or select it via the client application stream.
2. [cite_start]**Media Activation:** Mount the staged image utilizing the native **USBridge Client** application interface or trigger the mount directly through the appliance's front panel LCD and rotary encoder.
3. [cite_start]**Host Initialization:** The target server's hardware subsystem immediately enumerates a new USB mass storage device line[cite: 26]. [cite_start]This makes the bootable media instantly available within the native BIOS/UEFI Boot Menu for direct deployment[cite: 26].

---

## 3. High-Speed Performance & Integrity Caching

To compensate for the network latency inherent in streaming large images across different network topologies, the subsystem leverages an optimized data path:
* [cite_start]**LPDDR4X Cache Engine:** The appliance utilizes its high-speed internal RAM cache to absorb random read/write patterns generated by the host operating system during installation sequences. [cite_start]This fully compensates for network propagation delays and dramatically speeds up file copying[cite: 27].
* [cite_start]**Read-Write Overlay Mode:** When mounting an image in a writable state, the KVM redirects all new block modifications to a separate isolated layer, keeping your source "Golden Image" completely pristine and untouched[cite: 27].
